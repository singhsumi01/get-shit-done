**Vertical-slice planning mode — MVP audit + 5 new SDK verbs + test-rigor retrofit (#2826)** — closes the umbrella by addressing audit findings from a deep review of the merged Vertical MVP Slice feature. Fixes four functional defects: **D1** `docs/COMMANDS.md` user-story format mismatched the validator regex (slash separators vs the canonical comma form); **D2** Walking Skeleton triggered on brownfield projects because the predicate only counted prior summaries (now requires zero source files via the new `phase.walking-skeleton-trigger` SDK verb); **D3** the MVP+TDD gate's predicate logic was scattered across phase-mode resolution, tdd-mode lookup, behavior-adding check, and `git log` grep in workflow markdown (consolidated into the new `task.tdd-gate-check` SDK verb); **D4** no regression test pinned `**Mode:**` preservation across goal mutation. Closes three edge-case gaps: **E4** `**TDD:** true` in ROADMAP.md had no effect (now honored via the new `phase.tdd-mode` SDK verb with symmetric precedence chain `cli_no_flag → cli_flag → roadmap → config`); **E7** no `--no-mvp` opt-out flag for temporarily disabling mvp mode without editing ROADMAP.md; **E8** SKELETON.md was silently re-emitted on Phase 1 re-plan (now gated via the new `phase.skeleton-status` SDK verb with an explicit `--regenerate-skeleton` override). Adds five new SDK verbs total — the four above plus `workflow.parse` returning a typed IR (`sections`, `code_blocks`, `asks`, `references`, `sdk_calls`, `conditionals`, `bash_assignments`, `terms`) — used to retrofit eleven test files that previously source-grep'd workflow markdown into structural assertions on the typed IR (96 assertions total, 30 of them counter-tests for negative-space coverage). Also adds `workflow.mvp_mode` and `workflow.tdd_mode` to `SCHEMA_DEFAULTS` (lockstep across `config.cjs` and `config-query.ts`), removes six stale `intel.*` entries from golden-policy coverage lists left over from the Phase 6 CJS↔SDK seam migration, and adds the `--regenerate-skeleton` and `--no-mvp` / `--no-tdd` CLI flags with word-boundary parsing in `plan-phase.md`.
